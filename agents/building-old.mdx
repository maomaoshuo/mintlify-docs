创建 Dola Agent 是一个直接的流程。你可以从一个空白想法开始，逐步完成配置，并把它发布成市场中可购买的产品。本指南会按顺序带你完成每一步，从命名 Agent 到正式上架。第一次跟着做完后，之后创建 Agent 会变得很自然。

## 逐步创建并发布 Agent

<Steps>
  <Step title="打开 Creator Dashboard 并新建 Agent">
    前往 [dolaio.cn](https://dolaio.cn) 并登录创作者账号。在 Creator Dashboard 中点击 **New Agent**。系统会打开 Agent 编辑器，你将在这里完成发布前的所有配置。
  </Step>

  <Step title="设置名称并上传头像">
    给 Agent 一个清晰、好记，并能体现用途的名称。例如 “Legal Brief Assistant” 或 “Python Tutor”，能让买家在阅读描述前就知道自己将获得什么。

    上传头像图片，为 Agent 建立视觉身份。头像会出现在市场商品页、买家工作区和每个对话线程中。

    <Note>
      建议使用至少 256×256 px 的正方形图片，以保证 Agent 在所有界面中都有较好的显示效果。
    </Note>
  </Step>

  <Step title="编写简短公开描述">
    公开描述就是 Agent 的门面。它会显示在市场商品页，也是潜在买家首先阅读的内容。建议保持简洁，两到四句话即可，重点说明 Agent 做什么、适合谁、解决什么问题。

    <Tip>
      在公开描述中加入具体使用场景，帮助买家立即理解 Agent 的价值。例如：“使用这个 Agent 起草回复邮件、总结长对话，并建议后续行动；一个提示词即可完成。”
    </Tip>
  </Step>

  <Step title="编写系统提示词并定义 Agent 角色">
    系统提示词是 Agent 最重要的部分。它包含详细指令，定义 Agent 如何行动、知道什么、避免什么，以及如何沟通。买家期望 Agent 从第一条消息开始就表现正确，而系统提示词就是实现这一点的关键。

    一个好的系统提示词通常覆盖：
    - **角色和目标** — 这个 Agent 是什么？它的工作是什么？
    - **语气和沟通风格** — 正式还是轻松？简洁还是详细？
    - **知识范围** — 处理哪些主题？哪些请求应该拒绝？
    - **回复格式** — 使用要点、Markdown、短回答还是结构化输出？
    - **边界场景** — 遇到模糊或无关请求时如何处理？

    <Tip>
      系统提示词要清晰、具体。买家希望 Agent 开箱即用。模糊或过短的提示词容易带来不稳定回复和差评，发布前值得投入时间打磨。
    </Tip>
  </Step>

  <Step title="选择 AI 模型">
    选择驱动 Agent 推理和回答的 AI 模型。不同模型有不同强项，有的擅长对话和推理，有的擅长代码，有的适合创意任务。

    在 Agent 编辑器中打开 **Model** 下拉菜单，并选择最匹配 Agent 用途的模型。你可以查看 [模型页面](/agents/models-old)，了解如何按使用场景匹配模型。
  </Step>

  <Step title="添加 Skill（可选）">
    Skill 是模块化能力，可以让 Agent 做标准对话以外的事情。你可以在 Agent 编辑器的 **Skills** 标签页中添加 Skill。常见例子包括网页搜索、图像生成、文件处理和外部服务集成。

    添加 Skill 不是必须的，但拥有相关 Skill 的 Agent 通常能交付更高价值，而高价值 Agent 更容易销售。

    查看 [Skill 页面](/agents/skills-old)，了解可用 Skill 类别以及添加方式。
  </Step>

  <Step title="设置价格">
    选择买家如何为你的 Agent 付费。Dola 支持两种定价模式：

    <Tabs>
      <Tab title="一次性购买">
        买家支付一次，即可永久访问你的 Agent。这种模式适合交付明确、边界清晰的价值，例如专业写作助手、垂直研究工具或特定任务自动化。
      </Tab>
      <Tab title="订阅">
        买家按月或按年支付持续访问费用。这种模式适合买家会频繁使用的 Agent，例如每日效率助手或持续运行的客服工具。
      </Tab>
    </Tabs>

    <Note>
      Dola 会员套餐会为 Agent 提供初始使用额度。新买家可以先体验 Agent，再完全投入购买流程，从而降低购买门槛并提升转化率。
    </Note>
  </Step>

  <Step title="预览 Agent 并测试回复">
    发布前，使用内置预览测试 Agent。发送代表真实买家场景的示例消息，并检查：

    - Agent 是否按照系统提示词回复
    - 语气和格式是否符合公开描述
    - 边界场景和无关请求是否能被妥善处理
    - 如果添加了 Skill，Skill 是否能在合适上下文中正确触发

    反复调整系统提示词，直到 Agent 能稳定表现出你想要的行为。
  </Step>

  <Step title="点击 Publish 正式上线">
    当你对配置和测试结果满意后，点击 **Publish**。Agent 会立即出现在 Dola 市场中，并对该分类下的所有买家可见。

    发布后，你可以随时回到 Agent 编辑器，更新名称、头像、描述、系统提示词、模型、Skill 或价格。变更会立即对已有买家和未来买家生效。
  </Step>
</Steps>

## 发布之后

发布不是结束，而是开始。持续关注买家反馈和对话模式，理解用户实际如何使用你的 Agent。用这些信息优化系统提示词、添加新 Skill 或调整价格。Dola 上表现最好的创作者，都会把 Agent 当作持续迭代的产品。
