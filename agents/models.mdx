Dola 的 AI 模型页是统一的模型入口。你可以在这里查看可用模型与价格，创建和管理 API Key，阅读接入说明，并测试模型是否可用。

页面地址是 [shop.dolaio.cn/model](https://shop.dolaio.cn/model)。它面向两类使用方式：一类是在 Dola Agent 中选择模型；另一类是在你自己的私有场景中，通过 API Key 接入模型服务。

## 模型任选

Dola 不把 Agent 或开发者锁定在单一模型上。模型页会集中展示可用模型，方便你按任务选择合适能力。

<CardGroup cols={2}>
  <Card title="模型列表" icon="list">
    查看当前可用模型、模型供应方和价格信息。具体可用模型会随平台接入和模型提供商更新而变化。
  </Card>
  <Card title="模型切换" icon="shuffle">
    在 Agent 实例或模型选择入口中更换模型，让同一个 Agent 使用更适合当前任务的能力。
  </Card>
  <Card title="模型检测" icon="check">
    测试模型是否可用，适合在接入前确认模型响应和配置是否正常。
  </Card>
  <Card title="价格参考" icon="tag">
    查看不同模型的输入、输出或特殊能力价格，便于评估长期使用成本。
  </Card>
</CardGroup>

常见模型方向包括 GPT、Claude、DeepSeek、Gemini、Grok、Qwen、Kimi / Moonshot、Mistral、Llama 等。不同模型适合不同任务：有的适合推理，有的适合代码，有的适合低成本高频调用，有的适合图像生成或多模态任务。

## API Key 管理

模型页提供 **API Key** 管理能力。创建 API Key 后，你可以把它用于 Dola 的模型调用、私有应用接入、开发工具配置或 OpenClaw 网关接入。

<Steps>
  <Step title="打开 API Key">
    在 [shop.dolaio.cn/model](https://shop.dolaio.cn/model) 页面进入 **API Key** 区域。
  </Step>
  <Step title="新增 API">
    创建新的 API Key，并填写便于识别的名称。
  </Step>
  <Step title="复制并妥善保存">
    API Key 创建成功后请立即复制并保存。后续调用模型时，需要把它作为鉴权凭证使用。
  </Step>
  <Step title="按用途管理">
    你可以为不同项目、Agent、测试环境或客户场景创建不同 Key，方便后续排查和停用。
  </Step>
</Steps>

<Warning>
  API Key 等同于调用权限。不要把它写进公开仓库、截图、文档示例或前端代码中。
</Warning>

## 接入说明里有什么

模型页的 **接入说明** 会提供不同场景的调用示例。当前页面能看到的接入方向包括：

- **OpenAI 兼容接口** — 使用 `chat/completions`、`models` 等接口调用文本模型。
- **流式输出** — 通过 `stream: true` 获取持续返回的模型输出。
- **Anthropic 兼容调用** — 使用 Claude 类模型时，可参考 Anthropic 风格示例。
- **Codex 配置** — 在本地工具中配置 Dola 作为模型提供方。
- **图片生成与编辑** — 使用图像模型完成 `images/generations` 和 `images/edits` 类任务。
- **OpenClaw 接入** — 通过 `openclaw onboard` 配置自定义 Base URL、模型 ID、API Key 和兼容模式。

<Note>
  具体 Base URL、模型 ID 和示例代码请以 [shop.dolaio.cn/model](https://shop.dolaio.cn/model) 页面中的接入说明为准。模型和接口能力可能随平台更新调整。
</Note>

## 在 Agent 中选择模型

如果你只是使用或管理 Dola Agent，不一定需要手写 API。你可以直接在 Agent 实例页或模型选择弹窗中选择当前 Agent 使用的模型。

<Steps>
  <Step title="进入 Agent 实例页">
    打开已购买或已创建的 Agent，查看实例页中的 **模型选择** 卡片。
  </Step>
  <Step title="查看当前模型">
    页面会显示当前模型、价格信息和可用状态。
  </Step>
  <Step title="更换模型">
    点击 **更换模型**，从可选模型列表中选择更适合当前 Agent 的模型。
  </Step>
  <Step title="确认生效">
    保存后，Agent 会使用新的模型继续处理任务。模型调用费用会进入 AI 支付记录和模型 Token 消费明细。
  </Step>
</Steps>

## 如何选择

- **日常对话和客服** — 优先选择稳定、上下文表现好的模型。
- **代码、自动化和工具调用** — 选择代码能力或结构化推理更强的模型。
- **高频低成本任务** — 选择价格更低、响应更快的模型。
- **复杂推理任务** — 选择推理能力更强的模型。
- **图片生成或编辑** — 选择图像模型，并按图片尺寸、质量和任务类型评估费用。

<Tip>
  选择模型时不要只看单次价格。更合适的模型可能用更少轮次完成任务，长期成本反而更低。
</Tip>
